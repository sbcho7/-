# 📁 데이터 폴더 구조

## 폴더 구성

```
data/
├── channel/             ← 채널 로우 데이터
│   ├── 2025-01-01_channel.csv
│   ├── 2025-01-02_channel.csv
│   └── ...
└── appsflyer/           ← 앱스플라이어 로우 데이터
    ├── 2025-01-01_appsflyer.csv
    ├── 2025-01-02_appsflyer.csv
    └── ...
```

## 파일 네이밍 규칙

```
YYYY-MM-DD_channel.csv
YYYY-MM-DD_appsflyer.csv
```

- **날짜 형식**: YYYY-MM-DD (예: 2025-01-01)
- 규칙이 맞으면 **파일명이 달라도 자동 수집**됩니다 (폴더 내 모든 CSV 읽음)

## 컬럼 구조

### channel (채널 로우 데이터)
| 컬럼 | 설명 |
|------|------|
| 일 | 날짜 (YYYY-MM-DD) |
| 채널 | 구글 / 메타 / 네이버 |
| 채널분류 | 외부 / 자체 |
| 캠페인 | 캠페인명 |
| 캠페인목적 | 플러스가입 / 리타겟팅 / 첫구매 등 |
| 그룹 | 광고그룹명 |
| 소재 | 소재명 |
| 노출 | 노출수 |
| 클릭 | 클릭수 |
| 비용 | 광고비용 (원) |
| 회원가입 | 회원가입 전환수 |
| 구매 | 구매 전환수 |
| 구매매출 | 구매 매출 (원) |

### appsflyer (앱스플라이어 로우 데이터)
| 컬럼 | 설명 |
|------|------|
| 일 | 날짜 (YYYY-MM-DD) |
| 미디어소스 | googleadwords_int / Facebook Ads / naver_search 등 |
| 캠페인 | 캠페인명 (channel과 동일 기준) |
| 그룹 | 광고그룹명 |
| 소재 | 소재명 |
| 클릭 | AF 클릭수 |
| 회원가입 | AF 회원가입 전환수 |
| 구매 | AF 구매 전환수 |
| 구매매출 | AF 구매 매출 (원) |

## JOIN 구조

```
channel ──┐
          ├── JOIN (일 + 캠페인 + 그룹 + 소재) ──→ 통합 DataFrame
appsflyer ┘
```
- **LEFT JOIN** 기준: channel이 기준, AF는 매칭된 값만 추가
- AF 컬럼은 접두사 `AF_` 추가 (AF_클릭, AF_회원가입, AF_구매, AF_구매매출)

## 매일 업데이트 방법

1. 전날 데이터 파일 준비
   - `2025-01-02_channel.csv`
   - `2025-01-02_appsflyer.csv`
2. 각 폴더에 파일 넣기
3. 대시보드 **새로고침** 클릭 (또는 5분 대기)
